# 测试文档

> 最后更新：2026-07-20

## 可执行测试

项目使用单文件自测脚本，没有引入 Jest/Mocha 等测试框架。

完整自测：

```bash
cd sign-compose-app
npm test
```

也可以把自有 xlsx 作为兼容性测试输入：

```bash
cd sign-compose-app
node test/selftest.js '<xlsx路径>'
```

不传路径时，脚本会在操作系统临时目录生成全合成 xlsx、隔离测试数据和导出产物，不读取真实业务样例，也不写入应用用户目录。结束时会打印测试产物目录；脚本当前不会主动清理它。任一断言失败时命令以非零状态退出。

## 2026-07-20 实测结果

`npm test` 退出码为 0，70 个 `PASS`，没有 `FAIL` 或未捕获异常。全合成样例中检测到 1 张原有图片，连续两次导出都只增加 1 张签名图，用户的其他单元格编辑得到保留，源文件哈希保持不变。`npm audit --omit=dev` 报告 0 个已知漏洞。

发布前复检还确认：全部项目 JavaScript 可通过 `node --check`；workspace（排除 `node_modules`）没有表格、签名图片、密钥文件、私人绝对路径、邮箱、固定默认凭据或已知真实组织名称。应用源码中的唯一 `http://` 字符串是测试 SVG 的 XML namespace，不会发起网络请求。

## 覆盖范围

| 模块 | 已覆盖行为 |
|---|---|
| 用户与会话 | 双账号创建、目录隔离、公开 profile 脱敏、密码长度、正确/错误密码、账号大小写、重复账号、无隐式默认账号、会话切换失效、一次性导入授权与重放拒绝 |
| 资源边界 | 文件大小、无效 xlsx、压缩包条目数、声明解压体积、工作表行列与总格数上限 |
| 图像 | 二值化白底、笔画加粗、三字裁框、裁图白底、单字拼接、缺字字体混排 |
| 签名库 | 整词/单字读取、用户隔离、目录复制备份、多图、设默认、删默认、补单字、缩略图读取、新写入路径与旧索引读删路径的越界回归、文件名隐私回归 |
| xlsx 网格 | 行列尺寸、样式对象、合并区、已有嵌入图 |
| 合成布局 | 整词模式、合成阶段不写文件、96% 自动填充、等比、不越界、合并区感知 |
| 调整 | 手动自由拉伸、重生成恢复等比、左上对齐、删除签名 |
| 缺字 | `partial` 模式和具体缺字报告 |
| 导出 | 生成文件、清空目标文字、保留原图、新增签名图、连续导出不累加、保留其他单元格编辑、原文件哈希不变、默认文件名 |

## 未自动覆盖

- Electron 窗口内的真实点击、拖框、拖拽、右键菜单与缩放交互；
- 文件选择/保存对话框的取消和权限异常；
- Excel/WPS 的人工视觉一致性；
- 加密或包含复杂 Excel 特性的工作簿；资源超限拒绝逻辑已有模块测试，但未做超大真实工作簿压力测试；
- `library.json` 损坏、磁盘满、只读目录等 I/O 异常；
- macOS 版本兼容和生产安装包；
- 应用数据目录加密、系统账号被攻破后的攻击面。

> ⏳ 待补充：为 GUI、系统文件对话框和 I/O 权限异常增加自动化测试。
