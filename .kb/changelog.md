# 变更日志

> 最后更新：2026-07-20

格式按 Added / Changed / Deprecated / Removed / Fixed / Security 分类。项目此前没有正式发布日志，因此历史日期仅记录能从现有文件确认的内容。

## [未发布]

### Added

- 初始化 `.kb/` 项目知识库，包含导航、技术栈、IPC、数据结构、部署、开发规范、业务逻辑、测试、配置、FAQ 和变更日志。
- 新增面向公开仓库的根目录 `README.md` 和 `.gitignore`。
- 自测会在临时目录自动生成不含真实世界信息的 xlsx 测试样例。
- 新增可重复执行的 70 项本地自测和生产依赖审计流程。

### Changed

- `npm test` 现在直接运行完整自测，任一断言失败时以非零状态退出。
- 签名资源改用随机 UUID 作为 ID 和文件名，文件名不再包含姓名或词语。
- 使用 npm override 将 ExcelJS 的间接 `uuid` 依赖升级到 11.1.1。
- 导出改为基于当前内存工作簿快照创建副本，连续导出不会重复累加图片或丢失单元格编辑。

### Deprecated

> ⏳ 待补充：无已知弃用项。

### Removed

- 移除启动时自动创建的固定默认账号和登录页默认凭据提示。
- 移除项目文档中的真实组织名称与真实业务样例文件依赖。

### Fixed

- 修复签名词语可通过 `../` 影响其他账号目录内 PNG 的路径越界问题。
- 修复切换账号时异步文件操作可能跨越用户会话的竞态。
- 修复来自签名库或工作簿的动态文本可进入 HTML 的注入边界。

### Security

- 所有签名索引路径在读取、写入和删除前均验证位于当前账号目录内。
- 仓库忽略规则默认排除用户数据、表格、环境文件、私钥、日志、依赖与构建产物。
- 发布前隐私检查确认项目不再包含固定默认凭据或已知真实组织名称。
- `npm audit --omit=dev` 的已知漏洞数由 2 个中危依赖链告警降为 0。
- 主进程不再接受渲染进程提供的导入文件路径，并限制密码、图片、xlsx 压缩包和工作表资源范围。

## [1.0.0.0] - 2026-07-20

### Added

- Electron 桌面应用的登录、注册与按用户隔离的签名库。
- 签名照片逐字框选、二值化、笔画加粗、白边裁剪和整词/单字入库。
- 签名库搜索、拼音索引、排序、过滤、多图默认项与删除管理。
- xlsx 首工作表富网格展示、单元格编辑与已有图片显示。
- 整词优先、单字拼接、缺字系统字体混排三种合成模式。
- 签名移动、自由缩放、换图、重生成、对齐与删除。
- 另存 xlsx、保留原文件与原有嵌入图。
- `test/selftest.js` 模块/集成自测；历史基线为 46 项通过。

### Security

- BrowserWindow 启用上下文隔离并关闭 Node.js 集成。
- 密码使用随机盐和 scrypt 派生，登录比较使用 `timingSafeEqual`。

> 这是项目首次公开发布，暂无更早的公开版本历史。
