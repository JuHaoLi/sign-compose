# 常见问题与故障排查

> 最后更新：2026-07-20

## 怎么启动应用？

在终端运行：

```bash
cd sign-compose-app
npm ci
npm start
```

macOS 也可双击根目录的 `启动签名合成App.command`。

## 首次登录账号是什么？

没有预置账号或密码。首次启动后请在登录页选择“注册新账号”，设置自己的账号、昵称和 8 至 128 个字符的密码。

## 怎么运行测试？

运行：

```bash
cd sign-compose-app
npm test
```

测试会在操作系统临时目录创建全合成的账号、图片和 xlsx，不需要提供真实业务文件。只在排查特定兼容问题时才把自有测试文件路径作为 `node test/selftest.js <xlsx路径>` 的可选参数传入。

## 用户数据存在哪里？

主进程使用 Electron 的 `app.getPath('userData')/userData/users/`。每个用户一个独立目录，包含 `profile.json`、`library.json`、`chars/` 和 `words/`。

## 怎么备份签名库？

退出应用后，完整复制对应用户目录。不要只复制 `library.json` 或只复制 PNG，因为索引使用相对路径引用图片。自测已验证完整目录复制后仍可读取。

## 为什么签名合成结果用了系统字体？

这表示整词图不存在，并且至少一个单字也没有笔迹图。结果模式会标为 `partial`，界面会列出缺失的字。到“签名库”中直接补录该单字，或录入包含该字的整词签名，然后重新生成。

## 合成优先用整词还是单字？

优先找完全匹配的整词图；找不到才逐字拼接。整词和单字都有多张时，默认图优先；工作区右键“换一张字图”可以循环备选图。

## 为什么框选后提示没有有效墨迹？

裁剪区经过灰度与二值化后没有找到低于白边判断阈值的像素。重新框选并确保框内包含清晰笔画；过淡的照片也可能在阈值 180 的二值化中被当成背景。

## 能覆盖原始 xlsx 吗？

不能。保存路径与源文件相同时主进程会拒绝写入。默认输出名是 `原名_已合成.xlsx`。

## 为什么只看到第一个工作表？

`XlsxDoc.ws` 当前固定使用 `workbook.worksheets[0]`。多工作表选择尚未实现。

## 支持哪些输入文件？

- 签名图片：PNG、JPG/JPEG、WebP、BMP。
- 表格：`.xlsx`。

旧 `.xls`、CSV、加密 xlsx 和其他格式没有实现入口。

为避免本地文件耗尽应用内存，图片限制为 25 MB、8000 万像素；xlsx 限制为 50 MB，并对压缩包解压体积和首工作表范围设置上限。

## 表格会完全还原吗？

当前还原列宽、行高、合并区、常见字体/填充/对齐/边框和嵌入图片。但 exceljs 不保证支持 Excel 的所有高级特性。复杂条件格式、图表等需要针对真实文件人工验收。

## 应用是否联网？

源码中未发现网络请求；拼音、图像、用户和 xlsx 处理都在本地完成。依赖安装 `npm ci` 本身需要访问 npm registry，除非本机已有可用缓存。
